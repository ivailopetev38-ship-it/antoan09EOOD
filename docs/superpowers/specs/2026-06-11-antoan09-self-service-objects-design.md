# Самообслужване за обекти (Антоан-09)

**Дата:** 2026-06-11
**Статус:** одобрен от клиента

## Проблем
Клиентът (потребители на терен, не са технични) трябва сам да добавя данни.
Сегашно състояние:
- Нов **гасител в съществуващ обект** → лесно (сканиране → „➕ Добави в базата", или формата на картата на обекта). ✅
- Изцяло **нов обект/клиент** → само през импорта (`applyImport` създава клиенти/обекти), който **не е линкнат в менюто** и няма проста форма за единичен обект. ❌

## Цел
Клиентът сам да създава нови обекти/клиенти през приложението и да има лесен достъп до импорта — без програмист.

## Обхват (YAGNI)
- Форма създава САМО клиент+обект (гасителите се добавят после чрез сканиране/формата на обекта).
- Без редакция/триене на обекти на този етап.
- Без нови „бази" — една база (на Антоан); данните са Клиент → Обект → Гасители.

## Дизайн

### 1. Helper `findOrCreateSite` (нов, TDD)
`src/lib/import/site.ts` — `findOrCreateSite(db, { clientName, siteName, address?, phone? }) → { siteId, createdClient, createdSite }`.
- Намира клиент по име (case-trim); ако няма → създава.
- Намира обект по (клиент + име); ако няма → създава (с address/phone, ако са подадени).
- Идемпотентно. **`applyImport` се рефакторира да ползва същия helper** (без дублиран код).

### 2. API `POST /api/sites`
Приема `{ clientName, siteName, address?, phone? }`.
- Валидира задължителни `clientName`, `siteName` → иначе 400.
- Извиква `findOrCreateSite` → връща `{ ok, siteId }`. (Зад Basic/портал гейта.)
- `GET /api/sites` остава непроменен.

### 3. Страница `/obekti/nov`
Клиентска форма (стил като `/admin/import`): Име на обект, Клиент, Адрес (по избор), Телефон (по избор), бутон „Създай обект".
- При успех → пренасочва към `/obekt/{siteId}` (там веднага се добавят гасители).
- Мобилно: ползва глобалния form-стил (16px полета).

### 4. Навигация
В топбара на таблото добавям: **„🏢 Нов обект"** (→ `/obekti/nov`) и **„📥 Импорт"** (→ `/admin/import`).

## Тестване
Unit (`src/lib/import/site.test.ts`) за `findOrCreateSite`:
- нов клиент + нов обект → създава и двата;
- съществуващ клиент → нов обект под него (не дублира клиента);
- съществуващ клиент+обект → връща същия (идемпотентно);
- празни полета → грешка.
Плюс build + жива E2E проверка през Chrome (създаване на обект → отваря картата).

## Извън обхват
Редакция/изтриване на обекти, многофирмена изолация (multi-tenant), отделни бази.
